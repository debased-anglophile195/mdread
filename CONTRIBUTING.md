# Contributing to mdread

Thanks for wanting to make mdread better. It's a small, focused app and PRs are
welcome — bug fixes, accessibility improvements, typography polish, new themes.

## Principles

Keep these in mind; they're the whole point of the project:

- **Local-first & private.** Files are read in the browser and never uploaded.
  Don't add anything that phones home, loads a remote font, or sends data anywhere.
- **No runtime third parties.** The three markdown libraries (`marked`,
  `dompurify`, `highlight.js`) are bundled and served from our own origin. New
  runtime dependencies need a good reason.
- **System fonts only.** No web fonts — the reading experience uses the OS's
  native serif/sans.
- **It stays installable & offline.** Don't break the service worker / PWA shell.

## Getting set up

```bash
npm install
npm run dev          # Vite dev server with hot reload (http://localhost:5173)
```

Edit files under `src/` and the page reloads. To check a production build:

```bash
npm run build        # outputs ./dist
npm run preview      # serve ./dist exactly as deployed
```

## Project layout

```
index.html              app shell (Vite entry)
src/
  main.js               entry: imports styles, wires the UI, boots the app
  modules/              one concern per file (files, tree, editor, save, markdown, …)
  styles/               SCSS partials, assembled by main.scss
public/                 static assets copied verbatim (icons, manifest, og-image)
vite.config.js          build + PWA (service worker) config
wrangler.jsonc          Cloudflare Workers Assets (serves ./dist)
```

The app is plain ES modules + SCSS — no framework. Most behaviour lives in
`src/modules/`; the design lives in `src/styles/`. See `CLAUDE.md` for an
architecture tour.

### A couple of gotchas

- **Theme/typography tokens are CSS custom properties** (in `src/styles/_tokens.scss`)
  because the JS reads and writes them at runtime (`--reading-scale`, `--measure`,
  `--body`). Keep them as custom properties — don't convert them to Sass variables.
- The service worker is generated by `vite-plugin-pwa`; you don't hand-maintain a
  file list. If you change caching behaviour, edit the `VitePWA(...)` block in
  `vite.config.js`.

## Submitting a change

1. Fork and branch from `main`.
2. Make the change. Run `npm run build` to confirm it compiles, and
   `npm run preview` to sanity-check the real output in Chrome/Edge (live
   "save to disk" needs the File System Access API).
3. Keep the diff focused and match the surrounding code style.
4. Open a PR describing what changed and why. Screenshots help for anything visual.

There's no test suite yet; if you add one, even better. By contributing you agree
your work is licensed under the project's [MIT License](LICENSE).
