# Changelog

All notable changes to mdread are documented here. The format follows
[Keep a Changelog](https://keepachangelog.com/en/1.1.0/), and the project aims to
follow [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

### Changed

- **Build tooling.** Adopted a Vite + SCSS build. `npm run dev` now serves a
  hot-reloading dev server; `npm run build` outputs `./dist`, which is what
  Cloudflare serves. The deploy scripts build first.
- **Source layout.** The single `public/app.js` is now focused ES modules under
  `src/modules/`, and `public/styles.css` is SCSS partials under `src/styles/`.
  `index.html` moved to the repo root as the Vite entry. `public/` now holds only
  static assets (icons, manifest, OG image).
- **Markdown libraries.** `marked`, `DOMPurify`, and `highlight.js` are now real
  dependencies bundled into a `vendor` chunk instead of vendored `<script>` files.
  Still self-hosted — zero external requests at runtime.
- **Service worker.** Now generated by `vite-plugin-pwa` (Workbox) so it precaches
  hashed build assets automatically, replacing the hand-maintained file list.

### Added

- Contributor docs: `CONTRIBUTING.md`, `CODE_OF_CONDUCT.md`, `SECURITY.md`, GitHub
  issue and pull-request templates.

## [1.0.0]

### Added

- Initial release: local-first Markdown reader/editor. Open a file or folder,
  read with reading-tuned typography, edit in place (save to disk via the File
  System Access API, or download), Day/Sepia/Night themes, table of contents with
  scroll-spy, reading-progress bar, recent folders, installable offline PWA.
